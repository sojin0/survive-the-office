## 2. 사용성 테스트

### 2.1 테스트 방법

- **방식**: 모더레이티드 테스트 (진행자 1인 동석, 화면 녹화)
- **환경**: 데스크탑 Chrome 브라우저 / 모바일 Safari (각 절반씩)
- **소요 시간**: 1인당 약 20~30분
- **측정 도구**: 화면 녹화 + 구두 Think-Aloud + 종료 후 SUS 설문

### 2.2 테스트 태스크 및 시나리오

#### 시나리오 A — 하루 시작 & 이벤트 기록

> "오늘 출근해서 서비스에 처음 접속했습니다. 오전에 커피를 마시고, 갑작스러운 긴급 요청이 들어왔습니다. 이 두 가지를 기록해보세요."

| 태스크 ID | 태스크 | 성공 기준 | 목표 완료율 |
|---|---|---|---|
| T-01 | 서비스 접속 후 현재 HP와 날씨 상태 확인 | HP 수치와 날씨 아이콘을 5초 내 인지 | 100% |
| T-02 | "커피 한 잔" 긍정 이벤트 기록 | 버튼 클릭 후 HP 증가 및 툴팁 확인 | 90% |

| T-04 | 오늘의 이벤트 타임라인에서 기록 확인 | 두 이벤트가 타임라인에 순서대로 표시됨을 확인 | 85% |

### 2.4 SUS 설문지

> 각 문항: 1(전혀 그렇지 않다) ~ 5(매우 그렇다)

1. 이 서비스를 자주 사용하고 싶다
2. 이 서비스는 불필요하게 복잡하다
3. 이 서비스는 사용하기 쉽다
4. 이 서비스를 사용하려면 기술 전문가의 도움이 필요할 것 같다
5. 이 서비스의 다양한 기능이 잘 통합되어 있다
6. 이 서비스에는 일관성이 없는 부분이 너무 많다
7. 대부분의 사람들이 이 서비스를 빠르게 배울 수 있을 것이다
8. 이 서비스는 사용하기 매우 불편하다
9. 이 서비스를 사용하는 데 자신감을 느꼈다
10. 이 서비스를 사용하기 전에 많은 것을 배워야 했다

### 2.5 인터뷰 가이드 (테스트 후 5분)

**도입 질문**
- 오늘 처음 써봤는데, 전반적인 첫인상이 어땠나요?
- "HP"와 "날씨" 개념이 바로 이해됐나요?

**핵심 질문**
- 이벤트 버튼을 눌렀을 때 툴팁이 나타나고 사라지는 인터랙션이 어땠나요?
- HP가 변하는 걸 보면서 어떤 감정을 느꼈나요?
- 퇴근 결과 화면에서 가장 눈에 띈 건 무엇인가요?

**개선 질문**
- 가장 불편했거나 헷갈렸던 부분은 어디인가요?
- 있었으면 하는 기능이 있나요?
- 실제로 매일 쓸 것 같나요? 이유는?

### 3.1 시각적 일관성

| 항목 | 확인 기준 | 통과 여부 | 확인 방법 |
|---|---|---|---|
| 색상 토큰 | 모든 색상이 CSS 변수 사용, 하드코딩 없음 | ✅ | VSCode 전체 검색으로 하드코딩 색상값 없음 확인 |
| 날씨별 테마 | HP 범위에 따라 배경색/아이콘 정확히 전환됨 | ✅ | 브라우저에서 이벤트 클릭하며 6단계 전환 직접 확인 |
| HP 게이지 색상 | weatherState별 그라데이션 정확히 적용 | ✅ | HPGauge 코드 및 브라우저 실시간 확인 |
| 긍정/부정 이벤트 색상 | positive-bg / negative-bg 토큰 일관 적용 | ✅ | EventButton 렌더링 결과 육안 확인 |
| 폰트 패밀리 | Pretendard 전체 일관 적용 | ✅ | Chrome DevTools > Computed > font-family 확인 |
| 폰트 사이즈 | display/title/body/caption 토큰 정의 완료 | ✅ | tailwind.config.js + tokens.css 코드 확인 |
| 폰트 웨이트 | bold/medium/regular 토큰 정의 완료 | ✅ | tailwind.config.js + tokens.css 코드 확인 |
| 간격 | spacing 토큰(xs~2xl) 정의 및 적용 | ✅ | tokens.css 및 컴포넌트 className 코드 확인 |

## 4. 테스트 기반 개선 계획 및 결과

### 4.1 HistoryCalendar 반응형 개선 (🔥 High) — ✅ 완료

**문제:** 모바일 및 소형 모니터(13인치)에서 좌우 분할 레이아웃이 유지되어 우측 상세 패널이 잘리거나 스크롤이 필요함. T-08, T-09에서 오류 발생.

**개선 결과:** `flex-col md:flex-row` 분기 처리. 모바일에서 날짜 선택 시 상세가 캘린더 아래로 슬라이드-인. 360px 화면에서 T-08, T-09 오류 0회 달성.

### 4.2 초기 HP 80 시작 (💡 Medium) — ✅ 완료

**문제:** 테스트 참여자 다수가 "출근 자체가 이미 체력 소모인데 HP 100은 비현실적"이라는 의견 제시.

**개선 결과:** `INITIAL_HP` 100 → 80으로 변경. 출근·준비 과정의 기본 피로도 반영.

### 4.3 어두운 날씨 테마 텍스트 색상 미전환 (🔥 High) — ✅ 완료

**문제:** stormy·dead 배경에서 텍스트가 검정색 그대로 유지되어 가독성 저하.

**개선 결과:** MainLayout에 `weather-${weatherState}` 클래스 적용. CSS 변수 전파로 하위 모든 컴포넌트 텍스트 자동 전환.

### 4.4 팀보드 HP 실시간 반영 (🔥 High) — ✅ 완료

**개선 결과:** `mergedMembers`를 `useMemo`로 통합하여 `myHp`, `myWeather` 변경 시 즉시 재계산.

### 4.5 퇴근 후 대시보드 열람 기능 (💡 Medium) — ✅ 완료

**개선 결과:** `isViewingDashboard` 상태 추가. 퇴근 후 "대시보드 보기" 클릭 시 읽기 전용 모드로 열람 가능.

### 4.6 퇴근 확인 모달 추가 (💡 Medium) — ✅ 완료

**개선 결과:** HP별 게임스러운 멘트 + 확인/취소 플로우 추가. 실수로 퇴근 누르는 것 방지.
