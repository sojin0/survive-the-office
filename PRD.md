# PRD: 오늘 회사에서 살아남기 (Survive the Office)

> 버전: v1.0 | 작성일: 2025-03-13 | 작성자: UX/UI Designer, IT개발본부 UX유닛 | 상태: Draft

---

## 1. 문서 개요

### 1.1 목적
본 문서는 "오늘 회사에서 살아남기" 웹 서비스의 기획 의도, 기능 요구사항, UX 설계 방향, 기술 스택, 검증 계획을 명세한 Product Requirements Document(PRD)이다.

### 1.2 배경
직장인은 매일 크고 작은 업무 이벤트(회의, 긴급 요청, 메일 폭탄 등)를 처리하며 에너지를 소비한다. 반면 점심, 커피 타임, 팀 칭찬 등 긍정적 이벤트로 에너지를 회복하기도 한다. 그러나 이러한 감정 기복과 에너지 변화를 가시화하는 도구는 존재하지 않는다. 대부분의 근무 일지나 업무 툴은 딱딱하고 형식적이며, 유쾌하게 하루를 돌아보게 만드는 경험을 제공하지 못한다.

### 1.3 문제 정의
| 문제 | 설명 |
|---|---|
| 업무 감정의 비가시화 | 오늘 얼마나 힘들었는지, 얼마나 잘 버텼는지 수치로 볼 수 없다 |
| 기록 동기 부재 | 딱딱한 근무 일지는 지속적으로 작성하기 어렵다 |
| 긍정 경험 인식 부족 | 사소한 행복(커피, 칭찬)이 업무 지속력에 미치는 영향을 인지하지 못한다 |
| 팀 간 공감 부재 | 각자 얼마나 힘든지 유쾌하게 나눌 채널이 없다 |

## 2. 목표 (Goals & Success Metrics)

### 2.1 프로젝트 목표
- 직장인이 하루 업무를 **HP(체력 게이지)** 개념으로 유쾌하게 기록하고 돌아볼 수 있는 웹 서비스 제공
- 개인 기록과 팀 공유를 동시에 지원
- 긍정/부정 이벤트의 균형 있는 시각화로 번아웃 예방에 기여

### 2.2 성공 지표 (KPI)
| 지표 | 목표값 |
|---|---|
| 일 평균 이벤트 기록 수 | 1인 기준 3건 이상 |
| 퇴근 시 결과 확인율 | DAU 대비 60% 이상 |
| 팀 공유 기능 사용율 | 주간 활성 사용자 대비 40% 이상 |
| 재방문율 (D+1) | 50% 이상 |

## 3. 타겟 사용자 (Target Users)

### 3.1 주요 페르소나

**페르소나 A — 개인 기록형**
- 이름: 박지연 (30세, 기획팀 3년차)
- 특징: 하루 끝에 "오늘 왜 이렇게 힘들었지?" 싶은 날이 많음
- 니즈: 힘든 하루를 유쾌하게 정리하고 싶다. 일기보다 가볍고 빠른 기록 방식 선호
- 痛点: 기존 업무 일지는 너무 딱딱하고 형식적

**페르소나 B — 팀 공유형**
- 이름: 김민준 (34세, 개발팀 리드)
- 특징: 팀원들의 컨디션이 궁금하지만 물어보기 애매함
- 니즈: 팀 분위기를 가볍게 공유하고 서로 응원할 수 있는 채널
- 痛点: 슬랙/Teams는 너무 업무적이라 감정 표현이 어색함

## 4. 서비스 개요 (Service Overview)

### 4.1 서비스 컨셉
> "오늘 회사에서 살아남기"는 업무 이벤트를 HP(체력) 게이지로 표현하는 **직장인 생존 대시보드**다.
> 하루 동안 발생하는 다양한 이벤트를 기록하면 HP가 실시간으로 변동되고,
> 오늘의 날씨(업무 컨디션)가 대시보드에 시각화된다.
> 퇴근 시 "생존 퇴근" 또는 "전멸 퇴근" 결과를 확인하고 팀과 공유할 수 있다.

### 4.2 핵심 메타포
| 메타포 | 의미 |
|---|---|
| HP (체력 게이지) | 오늘 남은 업무 에너지. 0이 되면 "전멸" |
| 업무 날씨 | 현재 HP 상태를 날씨로 표현 (맑음 → 폭풍) |
| 이벤트 | HP를 증가/감소시키는 업무 행동 단위 |
| 퇴근 결과 | 하루 종료 시 HP 잔량 기반 생존 등급 판정 |

### 5.2 F-01: HP 게이지 대시보드

**설명:** 서비스 진입 시 가장 먼저 보이는 메인 화면. 오늘의 HP와 날씨 상태를 한눈에 확인.

**화면 구성 요소:**
- 오늘 날짜 및 요일 표시
- HP 게이지 바 (0~100, 시각적 색상 변화)
- 현재 업무 날씨 아이콘 + 상태 메시지
- 빠른 이벤트 입력 버튼 (하단 플로팅 또는 카드)
- 오늘 기록된 이벤트 타임라인

**HP ↔ 날씨 매핑:**
| HP 범위 | 날씨 | 아이콘 | 상태 메시지 |
|---|---|---|---|
| 81 ~ 100 | 맑음 ☀️ | 태양 | "오늘 무적 모드! 뭐든 가능해요" |
| 61 ~ 80 | 구름 조금 ⛅ | 구름+태양 | "순항 중. 이 기세 유지!" |
| 41 ~ 60 | 흐림 ☁️ | 구름 | "버티는 중. 커피 한 잔 어때요?" |
| 21 ~ 40 | 비 🌧️ | 빗방울 | "우산 챙기세요. 오늘 좀 빡세네요" |
| 1 ~ 20 | 폭풍 ⛈️ | 번개+비 | "전멸 직전! 간식이 필요한 순간" |
| 0 | 전멸 💀 | 해골 | "오늘은 여기까지… 수고했어요" |

### 5.3 F-02: 이벤트 기록

**설명:** HP에 영향을 주는 업무 이벤트를 빠르게 기록하는 기능.

**이벤트 카테고리 및 HP 변동값:**

#### ➕ 긍정 이벤트 (HP 증가)
| 이벤트 | HP 변동 | 설명 |
|---|---|---|
| ☕ 커피 한 잔 | +8 | 아메리카노의 기적 |
| 🍱 점심 맛있었음 | +12 | 오늘의 하이라이트 |
| 🍩 간식 등장 | +10 | 팀 간식은 팀워크 |
| 👏 칭찬 받음 | +15 | 인정욕구 충족 |
| ✅ 업무 완료 | +10 | 해냈다! |
| 😄 웃긴 일 있었음 | +7 | 소소한 즐거움 |
| 🏃 산책/스트레칭 | +8 | 몸도 마음도 리셋 |
| 💬 좋은 피드백 | +12 | 성장하는 느낌 |
| 🎉 프로젝트 성공 | +20 | 대형 이벤트! |
| 🏠 일찍 퇴근 예정 | +15 | 오늘 이긴 거 맞음 |

#### ➖ 부정 이벤트 (HP 감소)
| 이벤트 | HP 변동 | 설명 |
|---|---|---|
| 📧 메일 폭탄 | -10 | 읽씹하고 싶은 마음 |
| 🗓️ 긴급 회의 소집 | -12 | 갑툭튀 회의 |
| 😤 불필요한 회의 | -15 | 이건 메일로 해도 됐는데 |
| 🔥 긴급 요청 | -15 | 퇴근 30분 전 단골 손님 |
| 💻 시스템 오류 | -10 | 컴퓨터와의 전쟁 |
| 😶 피드백 없음 | -8 | 내 보고서 읽긴 한 건가요 |
| 🌀 야근 확정 | -20 | HP 대규모 손실 |
| 🤯 멀티태스킹 | -10 | 뇌가 과부하 중 |

**입력 UX:**
- 카테고리 탭으로 긍정/부정 구분
- 이벤트 버튼 클릭 → 클릭한 시각(HH:MM)이 타임스탬프로 자동 기록
- 버튼 클릭 시 해당 이벤트의 "설명" 텍스트가 툴팁 형태로 나타났다가 자동으로 사라지는 애니메이션 (예: "읽씹하고 싶은 마음 😤")
- HP 게이지가 변동값만큼 실시간 애니메이션으로 증가/감소
- 기록된 이벤트는 타임스탬프 + 이벤트명 + HP 변동값으로 오늘의 타임라인에 순서대로 누적
- 메모 옵션 (선택): 툴팁 노출 후 메모 입력창 선택적으로 열기 가능

### 5.4 F-03: 업무 날씨 표시

**설명:** HP 상태를 날씨 메타포로 시각화. 대시보드 상단에 항상 노출.

- HP 변동 시 날씨 아이콘 및 배경색 실시간 전환
- 날씨별 배경 그라데이션 적용
- 상태 메시지는 유쾌하고 공감가는 문구로 구성

---

### 5.5 F-04: 퇴근 결과 판정

**설명:** "퇴근하기" 버튼 클릭 시 오늘의 생존 등급과 요약을 표시.

**생존 등급표:**
| 등급 | HP 잔량 | 타이틀 | 코멘트 |
|---|---|---|---|
| S | 81~100 | 🏆 전설의 생존자 | "오늘 당신은 회사를 이겼습니다" |
| A | 61~80 | 😎 숙련된 생존자 | "흔들렸지만 끝까지 버텼어요" |
| B | 41~60 | 🙂 평균 생존자 | "오늘도 평타. 내일은 더 잘될 거예요" |
| C | 21~40 | 😮‍💨 간신히 생존 | "겨우 살아남았어요. 푹 쉬세요" |
| D | 1~20 | 🥵 기적의 생존 | "어떻게 버틴 건지 신기할 따름" |
| F | 0 | 💀 오늘은 전멸 | "수고했어요. 내일은 다를 거예요" |

**결과 화면 구성:**
- 오늘의 생존 등급 + 타이틀
- HP 시작(100) → 최저점 → 최종 잔량 요약 그래프
- 오늘 기록한 이벤트 TOP 3 (가장 큰 영향 이벤트)
- 팀 공유 버튼 (결과 이미지 공유 or 팀 보드 업데이트)

---

### 5.6 F-05: 팀 공유 보드

**설명:** 팀원들의 오늘 업무 날씨/HP 상태를 한 화면에서 확인.

**화면 구성:**
- 팀원별 카드: 이름/닉네임, 현재 날씨 아이콘, HP 잔량, 한마디
- 팀 전체 평균 날씨 표시 ("오늘 팀 전체 날씨: 흐림")
- 응원 리액션 기능 (👏 ☕ 🍩 등 이모지 반응)

---

### 5.7 F-06: 히스토리 캘린더

**설명:** 날짜별 생존 결과를 캘린더 형태로 시각화.

- 날짜 셀에 날씨 아이콘 표시
- 셀 클릭 시 해당 날 이벤트 상세 조회
- 주간/월간 평균 HP 트렌드 라인 차트

---

## 6. 화면 흐름 (User Flow)

```
[진입]
  ↓
[메인 대시보드] — HP 게이지 + 오늘 날씨 확인
  ↓
[이벤트 기록] — 긍정/부정 이벤트 탭 선택 → HP 실시간 반영
  ↓ (반복)
[대시보드 업데이트] — 날씨 변화 확인
  ↓
[퇴근하기] 버튼 클릭
  ↓
[퇴근 결과 화면] — 생존 등급 + 오늘의 요약
  ↓
[팀 공유] (선택) — 팀 보드에 결과 업데이트
  ↓
[히스토리] (선택) — 과거 기록 조회
```

### 7.5 접근성 기준

- 모든 색상 조합은 WCAG 2.1 AA 기준 충족 (대비율 4.5:1 이상)
- 포커스 상태: 모든 인터랙티브 요소에 2px 이상 포커스 링 적용
- 이모지/아이콘에 aria-label 또는 title 속성 부여
- HP 게이지: 색상 외 수치 텍스트 병행 표시 (색맹 사용자 고려)
- 키보드 네비게이션 지원: Tab 순서 논리적으로 구성

## 11. Lean Canvas

| **문제 (Problem)** | 1. 하루 업무의 감정적 에너지 소모를 가시화할 수단이 없다 2. 기존 근무 일지/업무 툴은 딱딱하고 형식적이라 지속 기록이 어렵다 3. 팀원의 컨디션과 업무 부하를 가볍게 공유할 채널이 없다 |
| **솔루션 (Solution)** | 업무 이벤트 버튼 클릭 한 번으로 HP가 실시간 변동되고, 오늘의 "업무 날씨"로 시각화. 퇴근 시 생존 등급 판정 + 팀 보드 공유로 공감과 연결 제공 |

| **가치 제안 (Unique Value Proposition)** | "오늘 회사에서 살아남았나요?" — 딱딱한 업무 기록을 유쾌한 생존 게임으로 바꿔, 기록이 즐거워지는 유일한 직장인 대시보드 |
| **채널 (Channels)** | 사내 슬랙/Teams 공유 링크 / 퇴근 결과 이미지 SNS 공유 (바이럴) / 사내 해커톤·사보 소개 / 팀 온보딩 시 소개 |
| **수익 구조 (Revenue Streams)** | (MVP 단계 무료) 향후: 팀 플랜 구독 (월정액) / 기업 라이선스 / 맞춤 이벤트 세트 커스터마이징 서비스 |
| **비용 구조 (Cost Structure)** | 프론트엔드 개발 및 유지보수 / Vercel 호스팅 비용 / (향후) 백엔드 서버·DB 비용 / 디자인 시스템 유지 |
| **핵심 지표 (Key Metrics)** | DAU(일간 활성 사용자) / 1인당 일 평균 이벤트 기록 수 / 퇴근 결과 확인율 / 팀 공유 기능 사용율 / D+1 재방문율 |
